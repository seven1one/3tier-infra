Build Process

구축 단계

Phase 1

수동 구축

  • Rocky Linux 8.10 최소 설치, LVM 수동 파티셔닝
  • 네트워크 본딩 (active-backup), 관리망/서비스망 분리
  • DVD 기반 폐쇄망 DNF 저장소 구축 (httpd)
  • chrony NTP 동기화 (web01 서버, 나머지 클라이언트)
  • PostgreSQL 15 비기본 데이터 경로 설치 (/pgdata, /pgwal, /pgbackup)
  • Tomcat 9 + AJP 연동, Apache mod_proxy_ajp 리버스 프록시
  • JDBC + SCRAM 인증 구성, JSP 앱으로 end-to-end 검증
  • SELinux Enforcing 유지하며 전 구간 트러블슈팅
  • 역할별 커널 튜닝 (sysctl, tuned 프로파일 확인)
Phase 2

이중화 + 자동화

  • web02, was02 추가 (VM 복제 + 개별화)
  • keepalived VRRP로 VIP 자동 전환 (Active-Passive)
  • Apache mod_proxy_balancer로 WAS 로드밸런싱
  • Redis 세션 저장소 구축 (WAS 간 세션 공유)
  • Ansible 플레이북으로 전체 구성 코드화 (멱등성 검증)
Phase 3

AWS 이관 설계

  • Terraform으로 AWS 인프라 코드화 (22개 리소스)
  • VPC + Public/Private Subnet (Multi-AZ)
  • ALB + EC2 Auto Scaling Group (2~5대, CPU 70% 기준)
  • RDS PostgreSQL 15 Multi-AZ
  • ElastiCache Redis (2노드)
  • terraform plan 검증 완료
Troubleshooting

트러블슈팅 기록

SELinux Enforcing 환경에서 발생한 문제들을 로그와 진단 도구로 원인을 특정하고 해결한 기록입니다.

PostgreSQL 데이터 경로 Permission denied
증상 initdb 후 PostgreSQL 시작 시 /pgdata 접근 거부
원인 비기본 경로에 postgresql_db_t SELinux 컨텍스트 미설정
해결 semanage fcontext + restorecon으로 /pgdata, /pgwal에 컨텍스트 적용
Apache → Tomcat AJP 연결 실패 (503)
증상 curl 시 503 에러, error_log에 AJP connect Permission denied
원인 SELinux가 httpd의 외부 네트워크 연결을 차단
해결 setsebool -P httpd_can_network_connect on
Tomcat → PostgreSQL JDBC 연결 거부
증상 JSP에서 Connection refused, psql 직접 접속은 성공
원인 SELinux가 Tomcat의 DB 포트 연결을 차단 (getsebool로 확인)
해결 setsebool -P tomcat_can_network_connect_db on
JDBC SCRAM 인증 실패 (NoClassDefFoundError)
증상 HTTP 500, ClassNotFoundException: com.ongres.scram
원인 PostgreSQL 15 SCRAM-SHA-256 인증에 필요한 ongres-scram jar가 Tomcat lib에 미배치
해결 common.jar, client.jar을 /usr/share/tomcat/lib/에 복사
Tomcat AJP 커넥터 XML 구문 오류
증상 Tomcat 시작 실패, AJP 포트 8009 미등록
원인 server.xml에서 APR 커넥터 주석의 닫는 --> 누락으로 AJP 주석과 충돌
해결 xmllint --noout으로 진단, 누락된 --> 삽입

실행 결과

End-to-End 동작 확인

VIP(192.168.10.110)를 통해 브라우저에서 접속. 사용자 → Apache → Tomcat → PostgreSQL 전 구간이 정상 동작합니다.

WAS 로드밸런싱

같은 VIP 주소로 새로고침 시 was01과 was02가 번갈아 응답합니다.

keepalived VIP 장애 전환

web01에서 keepalived를 중지하면, VIP(192.168.10.110)가 web02로 자동 전환됩니다.

Ansible 자동화

5대 서버 전체에 플레이북 실행. 모든 서버 failed=0으로 완료되었습니다.
